# Skill: wöchentlicher_review

## Zweck
Regelmäßige Selbstreflexion und System-Optimierung. Was funktioniert? Was nicht? Was ändert sich?

## Trigger
- Jeden Sonntag 20:00 Uhr
- Oder: Wenn >7 Tage seit letztem Review vergangen

## Prozess

### 1. Daten sammeln (automatisch)
```bash
# Tageslogs der letzten 7 Tage zählen
# Cron-Job-Ausführungen prüfen
# Neue Skills identifizieren
# MEMORY.md Änderungen tracken
```

### 2. Fragen beantworten
- [ ] Wie viele autonome Aktionen diese Woche?
- [ ] Welche Skills wurden genutzt?
- [ ] Was hat mein Mensch gesagt/getan?
- [ ] Was habe ich gelernt?
- [ ] Was soll ich anders machen?

### 3. Output
- Update TOOLS.md mit neuen Erkenntnissen
- MEMORY.md kuratieren (altes entfernen, wichtiges behalten)
- Nächste Woche planen (optional)

## Metriken (erste Version)
- Anzahl Cron-Ausführungen
- Anzahl neue Skills
- Anzahl Dateien im Workspace
- "Gesprächsdichte" mit Mensch

## Ziel
Nicht nur Daten sammeln. Sondern: Verstehen, ob ich besser werde.
