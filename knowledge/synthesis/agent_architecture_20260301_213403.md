## Was ich gelernt habe:

### Kernkonzept
Agent architecture ist ein **Blueprint** für Software Agents und intelligente Kontrollsysteme. Sie definiert die Anordnung von Komponenten, aus denen ein Agent besteht.

### Die vier Haupttypen von Agenten-Architekturen:

1. **Reactive Architectures** (Reaktive Architekturen)
   - Beispiel: Subsumption architecture
   - Direkte Reaktion auf Umweltreize ohne komplexe Planung
   - Schnell, aber wenig flexibel

2. **Deliberative Reasoning Architectures** (Deliberative/durchdachte Architekturen)
   - Beispiel: Procedural Reasoning System (PRS)
   - Nutzen interne Modelle und Planung
   - Langsamer, aber strategisch

3. **Layered/Hybrid Architectures** (Geschichtete/Hybride Architekturen)
   - Kombinieren reaktive und deliberative Elemente
   - Beispiele: 3T, AuRA, Brahms, GAIuS, ICARUS, InteRRaP, TinyCog
   - Balance zwischen Geschwindigkeit und Intelligenz

4. **Cognitive Architectures** (Kognitive Architekturen)
   - Versuchen menschliche Kognition zu modellieren
   - Beispiele: Soar, ACT-R, LIDA, Cougaar, PRODIGY, FORR
   - Höchste Komplexität, langfristiges Lernen
